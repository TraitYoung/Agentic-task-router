# ADR 001: 关于“身心痛感指标”量化标尺的校准决策

## 状态
已通过 (Accepted)

## 上下文 (Context)
在 Axiodrasil V15.0 的 MVP 测试阶段，发现大模型 (Qwen/DeepSeek) 在解析用户输入时，对 `pain_level` (1-10分) 的判定存在显著的“过度敏感”倾向。

**具体表现**：
用户输入“想哭”、“很累”等描述性词汇时，模型倾向于直接给出 7-8 分的高分，从而错误触发了红色医疗熔断逻辑（本意是处理心脏狂跳、手抖等物理濒危状态）。

## 决策 (Decision)
我们决定放弃单纯依赖 Pydantic 字段中的 `description` 进行约束，转而采用**“显式系统指令注入 (System Message Injection)”**的策略。

1.  **标尺硬化**：在 System Prompt 中显式定义 [1-3分]、[4-6分]、[7-10分] 的物理基准。
2.  **语义降级**：明确告知模型，所有夸张的描述（如“累死了”）若无具体生理症状支撑，严禁超过 6 分。
3.  **协议强制**：保留 Pydantic 的类型校验，作为数据的最后一道防波堤。

## 效果评价 (Consequences)
* **优点**：大幅提升了路由分发的准确性，成功区分了“求助情绪”与“生理崩溃”。
* **缺点**：增加了 Token 消耗，且对 System Prompt 的权重依赖度提高。
* **验证**：在“全军覆没想哭”的测试案例中，判定分值从 8 分降至 5 分，符合预期。
